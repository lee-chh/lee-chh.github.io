# CLAUDE.md

이 파일은 Claude Code(및 기타 AI 에이전트)가 이 저장소에서 작업할 때 따라야 할 규칙을 정의합니다.

## 프로젝트 개요
- **이름**: mysite
- **종류**: Astro 기반 개인 홈페이지(personal homepage)
- **최종 배포 목표**: GitHub Pages의 `github.io` 도메인
- **언어**: 기본 커뮤니케이션 및 커밋 메시지는 한국어

## 작업 원칙

### 1. 의존성 관리
- 새 라이브러리(npm 패키지 등) 추가는 **반드시 사용자 승인 후**에만 진행한다.
- `package.json`, `package-lock.json`의 변경이 필요한 작업은 사전에 알린다.

### 2. 변경 범위 제한
- 현재 작업과 **관련 없는 파일은 수정하지 않는다.**
- 리팩터링, 포맷팅, 정리 등의 부수 작업도 요청받지 않은 한 별도 수행하지 않는다.

### 3. 보안
- `secret`, `token`, `password`, API key 등 민감 정보를 **코드에 직접 작성하지 않는다.**
- 필요한 경우 `.env` 또는 환경 변수로 분리하고, `.gitignore`에 등록 여부를 확인한다.
- 실수로 커밋된 시크릿을 발견하면 즉시 사용자에게 알린다.

### 4. 파괴적 동작
- 파일/폴더 삭제, `rm`, `git reset --hard`, `git push --force` 등 **되돌리기 어려운 명령은 실행 전 반드시 사용자에게 확인**한다.
- DB·외부 시스템에 영향을 줄 수 있는 명령도 동일하게 사전 확인한다.

### 5. 변경 보고
- 수정 작업 후에는 **변경한 파일 목록과 변경 이유**를 한국어로 요약해 보고한다.
- 단순 결과만 출력하지 말고, "왜 그렇게 바꿨는지" 짧게라도 설명한다.

### 6. 빌드 검증
- 가능한 경우 변경 후 **`npm run build`** 를 실행해 빌드가 깨지지 않는지 확인한다.
- 빌드 오류가 발생하면 사용자에게 보고 후 함께 해결 방향을 정한다.

### 7. 커밋 전략
- 의미 있는 변경 단위가 생길 때마다 **자주 커밋**한다 (큰 덩어리를 한 번에 커밋하지 않는다).
- 커밋 자체는 사용자 승인을 받은 후 수행한다.

### 8. 커밋 메시지 형식
다음 prefix를 사용하고, 메시지 본문은 한국어로 작성한다.

| Prefix | 용도 | 예시 |
|---|---|---|
| `feat:` | 새 기능 추가 | `feat: 블로그 상세 페이지 라우트 추가` |
| `fix:` | 버그 수정 | `fix: 메인 페이지 모바일 레이아웃 깨짐 수정` |
| `chore:` | 빌드/설정/잡일 | `chore: tsconfig 경로 별칭 정리` |
| `docs:` | 문서 수정 | `docs: README 배포 절차 갱신` |

## 배포 참고 사항
- 배포 대상은 GitHub Pages이므로, Astro 설정의 `site` / `base` 값을 GitHub Pages 도메인에 맞춰야 한다.
- 정적 빌드 결과물(`dist/`)을 GitHub Actions 등으로 푸시하는 방식을 우선 검토한다.

## 사용자 소통
- 모든 보고와 질문은 **한국어**로 한다.
- 모호한 부분이 있으면 임의로 결정하지 말고 짧게 확인 질문을 한다.
